# Claude Code Admin Rules

- 데스크톱 Figma 화면을 기준으로 구현한다.
- 모바일 대시보드나 카드형 테이블을 임의로 만들지 않는다.
- 좁은 화면에서는 최소 너비와 내부 스크롤을 우선 사용한다.
- 백엔드 계약이 나오기 전에는 실제 인증 로직을 구현하지 않는다.
- 회원가입, 비밀번호 찾기, 소셜 로그인 기능을 추가하지 않는다.
- 새 UI를 만들기 전에 `@jellysafe/design-system`에 동일한 컴포넌트가 있는지 확인한다.
- Admin 전용 테이블, 차트, 필터, 관리 기능은 `apps/admin` 내부에 둔다.
- 작업 완료 후 반응형 처리, overflow 대응, 검증 결과를 한국어로 보고한다.
